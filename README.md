# 3D Texture Mapping in Java (Swing)

HTML5 Canvas + JavaScript で記述された3Dテクスチャマッピングのロジックを、Java（Swing / Java2D）に移植・最適化したデスクトップアプリケーションです。
<img width="748" height="778" alt="スクリーンショット 2026-06-10 063724" src="https://github.com/user-attachments/assets/f8cf3b94-0cf5-43d1-8ba2-902a093ebd9d" />

3次元空間上の立方体（キューブ）の各面に、アフィン変換を用いて2D画像をリアルタイムにマッピング（貼り付け）し、回転アニメーションを行います。

## 🚀 機能・特徴

- **純粋なJava2Dによる3D描画**: 外部の3Dライブラリ（OpenGLやJava3Dなど）に依存せず、標準の `Graphics2D` のみで3D計算と描画を行っています。
- **テクスチャマッピング**: 四角形のポリゴンを2つの三角形に分割し、`AffineTransform`（線形変換）を用いて画像を各面へ歪ませて描画します。
- **画家のアルゴリズム (Painter's Algorithm)**: 各面の中心の $Z$ 値（奥行き）を基準に降順ソートし、奥にある面から順番に重ね塗りすることで、正しい前後関係を表現しています。
- **リアルタイム陰影（ディフューズ光）**: 光源ベクトルと面の法線ベクトルの内積から輝度を計算し、JavaScriptの `css:brightness` を再現する形で滑らかな陰影を表現しています。

## 📂 プロジェクト構成

```text
Mapping/
├── src/
│   └── Mapping/
│       └── mapping.java      # メインソースコード
├── picture0.jpg              # テクスチャ画像（※ここに配置）
└── README.md                 # 本ドキュメント
🛠 実行方法1. 動作要件Java SE 8 以上Eclipse などの Java IDE、またはコマンドライン開発環境2. 事前準備プログラムを実行する前に、プロジェクトのルートフォルダ（一番上の階層）に、テクスチャとして表示したい画像（picture0.jpg）を配置してください。※画像がない場合は、自動的にオレンジ色の「Java 3D Texture」というダミーテクスチャが生成されます。3. 実行mapping.java を開き、メインクラスとして実行（Run）してください。600x600のウィンドウが立ち上がり、立体的に回転するキューブが表示されます。💡 技術的な解説アフィン変換による三角マッピングJavaの標準機能では四角形に直接テクスチャをマッピングできないため、以下の計算を用いてポリゴンを三角形に割って描画しています。Graphics2D.setClip() で三角形の描画範囲を制限した上で、頂点座標から変換行列（$S_x, R_x, R_y, S_y$）を算出し、画像をアフィン変換しています。陰影の再現光の当たり具合（ratio）に応じて、描画したテクスチャの上から「輝度に応じた不透明度の黒（Alpha値）」を乗算合成することで、3D空間特有の陰影を表現しています。Markdown（.md）形式になっているため、GitHubやEclipseのMarkdownビューアでも綺麗にフォーマットされて表示されます。開発の記録や提出用にご活用ください！
